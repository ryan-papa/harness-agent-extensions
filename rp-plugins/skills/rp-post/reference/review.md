# rp-post 독립 리뷰 프로토콜 (최종본 검수 SSOT)

사용자가 선택해 다듬은 **최종본 1개**를 **작성 에이전트가 아닌 독립 에이전트**가 평가한다. ⛔ 작성자 셀프리뷰 금지.
파이프라인 위치: 변형 선택·다듬기 직후, 최종 출력·저장 직전.

## 1. 원칙

| 규칙 | 내용 |
|------|------|
| 대상 | 선택된 최종본 1개만 (변형 전체 아님 — 변형은 체크리스트 셀프린트로 충분) |
| 독립성 | 리뷰어는 초안을 만든 에이전트와 분리. 작성 컨텍스트를 재사용하지 않는다 |
| 병렬 2인 | Claude·Codex 두 리뷰어에게 **동시에**(한 턴에 병렬) 요청. 서로의 결과를 모른 채 독립 평가 |
| 상한 2라운드 | R1 → 수정 → R2 → 수정 → 종료. R2 지적은 반영만, 추가 재요청 없음 |
| 조기 종료 | 어느 라운드든 두 리뷰어 모두 actionable 지적 0건이면 다음 라운드 생략 |
| 취합·판단 | 메인(작성) 에이전트가 두 리뷰를 병합. high·med 반영, low는 로그. 충돌 시 `patterns.md` 실측 데이터 우선, 채택/보류 근거 1줄 로그 |
| 비차단 | 한 리뷰어(특히 Codex CLI)가 불가하면 1줄 기록 후 가능한 리뷰어로 진행. 글 산출을 막지 않는다 |
| ⛔ 톤 보존 | 리뷰 반영이 사용자가 다듬은 개인 톤·표현을 획일화하지 않게 — 사실·구조 지적 위주로 반영 |

## 2. 리뷰어 호출

리뷰 전, 최종본과 메타를 임시 파일로 저장한다 (리뷰어가 읽을 수 있게):

```
<스크래치 디렉터리>/rp-post-review.md
— 내용: 최종본 전문 + 메타(아키타입·플랫폼·목표·소재 요약·사용한 체크리스트 종류)
```

| 리뷰어 | 방식 |
|--------|------|
| Claude | Agent 툴로 **새 서브에이전트** 생성. 위 파일 절대경로 + `archetypes.md`·`checklist.md`·`platforms.md` 경로 전달 |
| Codex | `codex exec "<프롬프트>"` 비대화형. 프롬프트에 위 파일·레퍼런스 절대경로를 명시하고 **파일을 읽고** 항목별 평가하도록 지시. 출력은 한국어 |

두 호출을 **같은 턴에 동시 실행**한다(Agent 툴 + Bash `codex exec` 병렬). 리뷰어에게 "너는 검수자다, 작성자가 아니다"를 명시한다.

## 3. 평가 항목 (리뷰어가 항목별로 반환)

| # | 항목 | 본다 |
|---|------|------|
| 1 | 사실 정확성 | 소재 대비 날조·과장·왜곡 없음, `[숫자 채우기]` 등 플레이스홀더 잔존, 수치·출처 정확 |
| 2 | 아키타입 공식 준수 | 선정 아키타입(`archetypes.md`)의 공식대로인가 — 훅/TL;DR 위치, 전개, 마무리 형태 |
| 3 | 체크리스트 독립 재채점 | 플랫폼에 맞는 체크리스트(`checklist.md` 바이럴/사내) 7항목을 **직접 다시 채점** — 작성자 채점과 다르면 지적 |
| 4 | 플랫폼 적합성 | 길이·형식·톤 (`platforms.md`): 쓰레드 500자·첫 150자 훅 / 슬랙 ≤300자+링크 / 사내 존댓말·두괄식 |
| 5 | 민감 정보 | 실명·시크릿·고객 데이터·내부 호스트/IP — **발견 즉시 high** |
| 6 | 독자 전달력 | 첫 줄에서 스크롤을 멈추는가(바이럴) / 3줄 안에 결론이 잡히는가(사내), 완독 가능성, 군더더기 문장 |
| 7 | AI 문체 냄새 | 기계적 병렬 구문·상투적 접속사·과잉 정중함 등 "AI가 쓴 티" — 사람이 쓴 글처럼 읽히는가 |

작성 에이전트가 이 항목 외에 필요하다고 판단한 기준을 추가해도 된다.

## 4. 반환 형식 (구조화)

각 지적: `{항목, 심각도(high/med/low), 위치(단락/문장), 문제, 수정안}`. 표 또는 JSON. 지적 없으면 "통과".

## 5. 루프

1. **R1** — Claude ∥ Codex 병렬 리뷰 → 두 결과 취합.
2. **수정** — 메인 에이전트가 병합·판단(충돌=`patterns.md` 실측 우선)해 최종본에 반영. high·med 반영, low 로그. 채택/보류 근거 1줄씩.
3. **R2** — 각 리뷰어에 **1회 더** 병렬 요청 → 취합 → 반영.
4. **종료** — R2 지적은 반영만. 남은 low 지적은 로그로 남기고, 리뷰 요약(라운드별 지적/반영 건수)을 최종 출력에 1~2줄 첨부.

로그 예: `채택[Codex-1]: 2문단 수치 출처 표기 추가 · 보류[Claude-6]: 첫 줄 축약 제안은 아키타입 A1 질문 훅 유지(patterns 실측 우선)`.
