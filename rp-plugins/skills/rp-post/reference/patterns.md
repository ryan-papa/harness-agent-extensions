## 데이터 갱신 절차 (수동, 필요 시)

1. 브라우저(gstack browse 등)로 threads.com 로그인 세션 확보
2. `threads.com/search?q=<키워드>` 키워드별 접속 → `window.scrollBy` 스크롤 반복하며 접근성 스냅샷 저장 (키워드당 4~5회)
3. 스냅샷에서 작성자·본문·좋아요·답글·리포스트·공유 파싱 (한글 단위 "천/만" 환산), 작성자+본문 앞 50자로 중복 제거
4. 좋아요 ≥5 필터 → 4분위 분리 → 위 피처 출현율 재계산
5. 본 파일의 표·사례·수집 일자를 갱신하고, 변화가 큰 항목은 `archetypes.md`·`checklist.md`에 반영
