# checklist.md — 초안 검증

변형마다 아래 표를 채워 첨부한다. 판정: ✓ 충족 / △ 부분 충족 / ✗ 미충족. **✗가 1개라도 있으면 그 변형을 수정 후 재검증** (의도적 예외는 사유 명기).

체크리스트는 둘 — **바이럴용**(A계열, SNS)과 **사내 공유용**(K계열, company). 플랫폼에 맞는 것 하나만 쓴다.

## 체크리스트 (바이럴 — A계열)

| # | 항목 | 판정 기준 | 실측 근거 |
|:---:|------|------|------|
| 1 | 첫 줄 훅 | 첫 줄 20~40자에 핵심 반전·정체성이 들어갔는가. 배경 설명으로 시작하면 ✗ | 첫 150자가 알고리즘 판단 구간 |
| 2 | 숫자 증명 | 자격·성과가 측정값으로 표현됐는가 ("15년 경력", "앱 30개"). 형용사뿐이면 ✗ | 숫자 성과 +6%p, 상위권 공통 |
| 3 | 1인칭 경험 | "~하는 법"이 아니라 "내가 ~해보니"인가. 정보 나열형이면 △, 목적이 저장용(A5)이면 예외 | 1인칭 스토리 훅 +22%p |
| 4 | 열린 마무리 | 질문·나눔 약속·행동 촉구로 끝나는가. 자기 완결로 닫히면 △ | 질문형 마무리 +8%p, ↩912 사례 |
| 5 | 길이 적정 | 스낵(≤80자) 또는 스토리(300~600자)인가. 어중간한 80~150자면 △ | 300~600자 중앙값 2배, 80~150자 최약 |
| 6 | 문장부호 절제 | 느낌표 연발·이모지 과다가 없는가 | 느낌표 −8%p (하위권 신호) |
| 7 | 지표-포맷 일치 | 사용자의 목표 지표와 아키타입이 매트릭스상 일치하는가 | patterns.md 지표별 매트릭스 |

## 출력 예시

| # | 항목 | 판정 | 코멘트 |
|:---:|------|:---:|------|
| 1 | 첫 줄 훅 | ✓ | "퇴사 후 한 달, 앱 30개" — 27자 반전 훅 |
| 2 | 숫자 증명 | ✓ | 30개·15년·3사 경력 |
| 3 | 1인칭 경험 | ✓ | 실험 서사 |
| 4 | 열린 마무리 | △ | 연재 예고는 있으나 질문 없음 → 목표가 답글이면 질문 추가 권장 |
| 5 | 길이 적정 | ✓ | 340자 스토리 구간 |
| 6 | 문장부호 절제 | ✓ | 느낌표 0 |
| 7 | 지표-포맷 일치 | ✓ | 목표=좋아요, A2 영수증 인증형 |

## 체크리스트 (사내 공유 — K계열)

| # | 항목 | 판정 기준 |
|:---:|------|------|
| 1 | 두괄식 TL;DR | 첫 3줄 안에 결론·핵심이 요약됐는가. 배경부터 시작하면 ✗ |
| 2 | 맥락 제공 | 해당 업무를 모르는 동료도 따라올 수 있는가 — 전제·용어·링크가 갖춰졌는가 |
| 3 | 재현 가능성 | 버전·명령·설정·에러 메시지 원문이 그대로 실려 따라할 수 있는가 (K1·K2) |
| 4 | 근거·출처 | 주장에 실측 데이터·공식 문서·PR 링크가 붙었는가. 추측은 추측이라고 표시했는가 |
| 5 | 다음 액션 | 읽은 사람이 무엇을 하면 되는지 명확한가 (적용 방법·문의처·후속 일정) |
| 6 | 존댓말·건조한 톤 | 존댓말인가. 과장·어그로·이모지 연발이 없는가 |
| 7 | 민감 정보 | 시크릿·고객 데이터·인사 정보·내부 호스트/IP가 없는가. **하나라도 있으면 즉시 ✗** |

## 흔한 실패 패턴 (초안에서 걸러낼 것)

| 패턴 | 왜 실패하는가 | 교정 | 적용 |
|------|------|------|:---:|
| 배경부터 서술 ("요즘 AI가 화두인데요…") | 훅 없이 첫 150자 소진 | 결과·반전을 첫 줄로 끌어올림 | A |
| "~하는 법 N가지" 제목형 | 리스트형 −14%p, 피드에서 스킵됨 | 같은 내용을 "내가 해보니" 경험담으로 | A |
| 겸손 완충 ("부족하지만", "별건 아니지만") | 자격 신호 약화 | 삭제하고 숫자로 대체 | 공통 |
| 마무리 자기 완결 ("도움이 됐길 바랍니다") | 대화 차단 → 답글 0 | 질문 또는 다음 편 예고로 교체 | A |
| 느낌표·이모지로 흥분 연출 | 하위권 신호 | 내용(숫자·반전)으로 흥분을 만든다 | 공통 |
| 시간순 일지 ("먼저 ~했고, 그다음 ~했고") | 결론이 안 보여 이탈 | TL;DR 먼저, 과정은 뒤에 (두괄식) | K |
| "재시작하니 됐습니다"로 종결 | 원인 없는 해결은 재사용 불가 | 원인 규명까지 쓰거나 "원인 미상"을 명시 | K |
| 일반론 요약 (공식 문서 번역) | 우리 맥락이 없어 남의 글 | "우리 프로젝트에서는"으로 연결 | K |
