# rp-deck 독립 리뷰 프로토콜 (생성 후 검수 SSOT)

생성·보강한 장표를 **작성 에이전트가 아닌 독립 에이전트**가 평가한다. ⛔ 작성자 셀프리뷰 금지.
파이프라인 위치: 자가 린트([`design-rules.md`](design-rules.md) §10) 직후, push(적재) 직전.

## 1. 원칙

| 규칙 | 내용 |
|------|------|
| 독립성 | 리뷰어는 장표를 만든 에이전트와 분리. 작성 컨텍스트를 재사용하지 않는다 |
| 병렬 2인 | Claude·Codex 두 리뷰어에게 **동시에**(한 턴에 병렬) 요청. 서로의 결과를 모른 채 독립 평가 |
| 상한 2라운드 | R1 → 수정 → R2 → 수정 → 종료. R2 지적은 반영만, 추가 재요청 없음 |
| 조기 종료 | 어느 라운드든 두 리뷰어 모두 actionable 지적 0건이면 다음 라운드 생략 |
| 취합·판단 | 메인(작성) 에이전트가 두 리뷰를 병합. 충돌 시 [`design-rules.md`](design-rules.md) 우선, 채택/보류 근거 1줄 로그 |
| 비차단 | 한 리뷰어(특히 Codex CLI)가 불가하면 1줄 기록 후 가능한 리뷰어로 진행. 장표 산출을 막지 않는다 |

## 2. 리뷰어 호출

| 리뷰어 | 방식 |
|--------|------|
| Claude | Agent 툴(Task)로 **새 서브에이전트** 생성. 대상 HTML 절대경로 + [`design-rules.md`](design-rules.md) 경로 전달 |
| Codex | `codex exec "<프롬프트>"` 비대화형. 프롬프트에 HTML 절대경로·`design-rules.md` 경로를 명시하고, **두 파일을 읽고** 항목별로 평가하도록 지시. 출력은 한국어 |

두 호출을 **같은 턴에 동시 실행**한다(Agent 툴 + Bash `codex exec` 병렬). 리뷰어에게는 "너는 검수자다, 작성자가 아니다"를 명시한다.

## 3. 평가 항목 (리뷰어가 항목별로 반환)

| # | 항목 | 본다 |
|---|------|------|
| 1 | 내용 정확성 | 원문 대비 사실 오류·누락·왜곡, 근거·수치 정확 |
| 2 | 구조·전달력 | 결론 먼저, 한 장 한 메시지, 주장형 헤드라인, 긴장→해소 흐름 |
| 3 | 시각 요소 | 추세·구성비·비교·구조에 적절한 차트/다이어그램 존재, 정직한 축, **설명 텍스트 병기** 유지 |
| 4 | 디자인·일관성 | 디자인 토큰·시각 위계·대비·여백, 장간 일관 |
| 5 | 토스 톤 | 해요체·문장형·긍정형, 명사 라벨·명사형 종결 헤드라인 금지 |
| 6 | 부가 설명·용어 | 낯선 용어 각주 충분·명료(약어 풀텍스트), 용어집, 부가 설명이 이해를 돕는지 |
| 7 | 자가 린트 | [`design-rules.md`](design-rules.md) §10 체크리스트 위반 여부 |

작성 에이전트가 이 항목 외에 필요하다고 판단한 기준을 추가해도 된다.

## 4. 반환 형식 (구조화)

각 지적: `{항목, 심각도(high/med/low), 슬라이드번호, 문제, 수정안}`. 표 또는 JSON. 지적 없으면 "통과".

## 5. 루프

1. **R1** — Claude ∥ Codex 병렬 리뷰 → 두 결과 취합.
2. **수정** — 메인 에이전트가 병합·판단(충돌=`design-rules.md` 우선)해 HTML에 반영. 채택/보류 근거를 1줄씩 로그.
3. **R2** — 각 리뷰어에 **1회 더** 병렬 요청 → 취합 → 반영.
4. **종료** — R2 지적은 반영만. 남은 low 지적은 로그로 남기고 종료.

로그 예: `채택[Codex-3]: 5장 추세를 표→선차트 · 보류[Claude-1]: 헤드라인 교체는 원문 근거상 현행 유지(design-rules §2 우선)`.
